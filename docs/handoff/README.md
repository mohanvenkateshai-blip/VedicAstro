# Handoff Documentation

This folder contains the machine-maintained artifacts that let any new AI (or human) take over the VedicAstro project instantly.

## Files
- `context.md` — **Latest session handoff snapshot** (2026-06-30). Read this first after a Cursor/software update or AI tool switch. Human-maintained; updated at milestone handoffs.
- `AI_TAKEOVER_PACK.md` — Single-file summary of live state, registered engines, refresh commands, and critical paths. **Always declares the mandatory multi-agent protocol.**
- Generated by `scripts/handoff/maintain_context.py`

## How It Stays Fresh
After any Knowledge Graph ingestion (`merge --promote`) or new engine registration:
```bash
python scripts/handoff/maintain_context.py --update-all
```
This script:
1. Introspects the live `KnowledgeEngine` + `EngineRegistry`
2. Updates marked sections in `CONTEXT.md`
3. Regenerates `AI_TAKEOVER_PACK.md`

**Multi-agent protocol is now part of the handoff contract:** every regeneration injects a prominent declaration and a "Multi-Agent Health" section, and lists `.cursor/rules/multi-agent-mandatory-protocol.mdc` as a critical file. All primary status documents (CONTEXT, STATUS, knowledge-engine-status, KNOWLEDGE_CATALOG) carry the same declarations and sections.

The system detects:
- New registered engines (via registry)
- Graph version / node count changes
- Health status of the store

## Multi-Agent Launch Infrastructure
For autonomous enforcement of the "start with 5+" rule on post-reorg work (embeddings, Supabase apply, visualizer, admin CLI, verification, etc.):

- Reusable launch blocks, focused subagent prompts, and the minimal drift-prevention checklist live in `docs/multi-agent-launch-templates.md`.
- Main agent (and any orchestrator sub-agent) should read this file at the first sign of substantial work and emit the ready-to-paste 8+ `Task` block as the opening action.
- The templates are designed to be pasted verbatim (with light tailoring of the focused prompts) and integrate with the permanent rule in `.cursor/rules/multi-agent-mandatory-protocol.mdc`.
- New waves should record themselves (agent count, types, links) and keep status files + takeover artifacts updated with "Multi-Agent Health" notes.
