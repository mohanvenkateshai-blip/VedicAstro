# Handoff Documentation

This folder contains the machine-maintained artifacts that let any new AI (or human) take over the VedicAstro project instantly.

## Files
- `AI_TAKEOVER_PACK.md` — Single-file summary of live state, registered engines, refresh commands, and critical paths. **Always declares the mandatory multi-agent protocol.** Start here.
- Generated by `scripts/handoff/maintain_context.py`

## How It Stays Fresh
After any Knowledge Graph ingestion (`merge --promote`) or new engine registration:
```bash
python scripts/handoff/maintain_context.py --update-all
```
This script:
1. Introspects the live `KnowledgeEngine` + `EngineRegistry`
2. Updates marked sections in `CONTEXT.md`
3. Regenerates `AI_TAKEOVER_PACK.md`

**Multi-agent protocol is now part of the handoff contract:** every regeneration injects a prominent declaration and a "Multi-Agent Health" section, and lists `.cursor/rules/multi-agent-mandatory-protocol.mdc` as a critical file. All primary status documents (CONTEXT, STATUS, knowledge-engine-status, KNOWLEDGE_CATALOG) carry the same declarations and sections.

The system detects:
- New registered engines (via registry)
- Graph version / node count changes
- Health status of the store
